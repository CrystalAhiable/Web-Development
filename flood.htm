ESRI ArcGIS Enterprise and ArcGIS Online provide extensive functionality to accomodate sophisticated web mapping techniques. In this section, we are demonstrating how these functionalities come to play for visualizing flood impact analysis using data published on the ArcGIS Server and displayed using StoryMaps and the Web AppBuilder. The section above is a Story Map created with ArcGIS StoryMaps. It offers a variety of opportunitites for creating interactive maps. For instance, with the swipe functionality, a different level of visualization can be achieved. In the scenrio where we compare the postcodes by their flood risk levels with the rainfall patterns, the swipe option allows us to view both layers side by side and this functionality gives more meaning to the layers in view. Some additional functionality with StoryMaps is the abillity to embed videos and images to enrich the story. In this example, a YouTube video on flooding in the Isle of Wight is embeded to demonstrate this functionality. Aside StoryMaps, another powerful tool introduced by ESRI is the Web AppBuilder as demonstrated below. In this scenario we are able to interact with the map better and derive meaningful information. For instance by clicking on the map, we are provided with more information about the features. Also, some added functionality like the 'info summary' at the bottom left corner provides summaries based on the information required. In this example, we use this functionality to give a summary on the number postcodes that are floodrisk. At the top left corner in the body of the map, users will be able carry out activities such as changing basemap layer, interact with charts for information such as rebuilding costs and floodrisks, search addresses for information about their flood risk level, change the layers in view and filter items. With ArcGIS, there are numerous possibilities available for visualizing geographic data, making it a product of choice. However, since it is not an open source software, a license is required to be able to access its plethora of functionalities making it a costly approach.
